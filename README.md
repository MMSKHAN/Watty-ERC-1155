WTY Energy Settlement System

A blockchain-based energy reporting and settlement system built on Ethereum using Solidity, Hardhat, and React.

This project demonstrates how energy production data can be verified and converted into digital energy credits using smart contracts. The system records meter activity, reports energy production, mints energy credits, and finalizes settlement events in a transparent and auditable way.

System Roles
Admin

System administrator.

Permissions:

register energy meters

assign or remove minters

Minter

Authorized backend system responsible for:

reporting energy production

minting energy credits

finalizing settlements

User

Regular participants who can:

hold energy credits

burn credits when redeemed

Contract Functions
setMinter

Assign or remove a minter role.

setMinter(address account, bool value)

Only callable by admin.

registerMeter

Registers a new energy meter.

registerMeter(bytes32 meterId, address owner, bytes32 metadataHash)

Parameters

Parameter	Description
meterId	unique identifier of meter
owner	wallet address of meter owner
metadataHash	reference hash for off-chain meter data
reportEnergy

Records energy production from a meter.

reportEnergy(bytes32 meterId, bytes32 intervalId, uint256 whDelta, bytes32 dataHash)

Parameters

Parameter	Description
meterId	registered meter id
intervalId	reporting interval
whDelta	produced energy in Wh
dataHash	hash of off-chain data
mintCredits

Mints ERC1155 energy credits.

mintCredits(bytes32 meterId, address to, uint256 tokenId, uint256 amount, bytes32 ref)

Credits represent verified energy units.

burnCredits

Burns energy credits when redeemed.

burnCredits(address from, uint256 tokenId, uint256 amount, bytes32 ref)

Callable by:

credit owner

authorized minter

finalizeSettlement

Stores settlement summary.

finalizeSettlement(
    bytes32 siteId,
    bytes32 period,
    uint256 totalWh,
    uint256 totalTokens,
    bytes32 proofHash
)

Emits a settlement event for auditing.

Events

The contract emits the following events:

MeterRegistered
EnergyReported
CreditsMinted
CreditsBurned
SettlementFinalized

These events enable auditability and external indexing.

Installation
1 Install Node.js

Node version 18 or higher.

node -v
Hardhat Setup

Navigate to Hardhat directory.

cd Hardhat

Install dependencies.

npm install
Compile Contracts
npx hardhat compile

Artifacts will automatically be exported to the UI project.

paths: {
  artifacts: "../UI/src/artifacts"
}

This allows the React UI to directly consume the contract ABI.

Run Local Blockchain

Start Hardhat local network.

npx hardhat node

RPC endpoint

http://127.0.0.1:8545

Chain ID

31337
Deploy Contract

Example deployment command:

npx hardhat run scripts/deploy.js --network localhost

Security Design

The system uses role-based access control.

Admin controls configuration.

Minters represent trusted backend services.

Meters must be registered before reporting energy.

Devices cannot mint tokens directly, preventing unauthorized token creation.
